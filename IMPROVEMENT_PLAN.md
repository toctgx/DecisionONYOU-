# DecisionONYOU 개선 계획서

## 목표
DecisionONYOU를 단일 HTML 기반 로컬 프로토타입에서, 유지보수 가능한 구조와 실제 서비스 전환이 가능한 형태로 정리한다.

## 현재 상태 요약
- `index.html` 한 파일에 HTML/CSS/JavaScript가 모두 포함되어 있음
- 데이터 저장은 `localStorage` 기반
- 질문 등록, 투표, 댓글, 신고, 관리자 화면까지 프로토타입 기능이 구현되어 있음
- 실제 멀티유저 서비스, 인증, 서버 권한 분리는 아직 없음

## 단계별 개선 방향

### 1단계 — 구조 진단 및 개선 계획 수립
목표: 현재 기능과 구조를 문서화하고, 무엇을 유지하고 무엇을 바꿀지 결정한다.

세부 작업:
- 현재 앱의 핵심 사용자 흐름 정리
- 프로토타입 기능 목록 정리
- 유지할 기능 / 제거 후보 기능 분류
- 기술 부채와 위험 요소 명시
- 리팩터링 우선순위 결정

산출물:
- 현재 구조 진단 보고서
- 단계별 실행 로드맵
- MVP 범위 재정의 초안

### 2단계 — 프론트엔드 구조 분리
목표: `index.html` 단일 파일 구조를 최소한의 유지보수 가능한 구조로 분리한다.

세부 작업:
- HTML / CSS / JavaScript 분리
- 기존 동작 최대한 유지
- 추후 모듈화가 쉬운 형태로 파일 정리
- 필요 시 assets 디렉터리 정리
- 로컬 실행 기준으로 동작 검증

권장 결과 구조 예시:
- `index.html`
- `styles.css`
- `app.js`

선택 확장:
- `js/storage.js`
- `js/ui.js`
- `js/admin.js`
- `assets/`

### 3단계 — 실서비스 전환 기술 방향 결정
목표: 이 앱을 실제 서비스로 키우기 위한 스택과 아키텍처 방향을 정한다.

비교 후보:
- Firebase
- Supabase
- Next.js + custom backend

검토 기준:
- 개발 속도
- 인증 구현 난이도
- 이미지 업로드 처리
- 실시간성
- 운영 난이도
- 비용
- 팀타덕의 개발 스타일과 유지보수 부담

결과물:
- 추천 스택 1안
- 대안 1~2안
- 왜 그 선택이 적합한지 설명
- 이후 개발 순서 제안

## 핵심 문제와 개선 원칙

### 핵심 문제
1. 단일 파일 구조로 인한 유지보수 어려움
2. `localStorage` 기반이라 실제 사용자 간 데이터 공유 불가
3. 인증/권한이 실제로 분리되지 않음
4. 관리자 기능이 클라이언트에 노출되어 있음
5. 앱 규모 확장 시 구조적 한계가 큼

### 개선 원칙
1. 먼저 구조를 분리하고, 그 다음 백엔드를 붙인다
2. 프로토타입 기능을 모두 살리기보다 핵심 MVP를 선명하게 만든다
3. 운영 기능은 프론트가 아니라 권한 있는 서버/DB 레이어로 옮긴다
4. 빠른 배포보다 유지보수 가능성을 우선 확보한다

## MVP 재정의 제안
실서비스 전환 시 1차 MVP는 다음 핵심 흐름 위주로 정리하는 것이 좋다.
- 회원가입 / 로그인
- 질문 등록 (이미지 포함)
- 피드 조회
- BUY/PASS 또는 THIS/THAT 투표
- 댓글
- 신고

보류 가능 기능:
- 과한 관리자 화면 세부 기능
- 복잡한 통계
- 세부 금지어 운영 UI
- 로컬 전용 세션/큐 로직 일부

## 실행 우선순위
1. 현재 구조 문서화 및 계획 확정
2. 프론트 코드 분리
3. UI/상태 구조 정리
4. 백엔드/DB/Auth 스택 선택
5. 실서비스형 데이터 모델 설계
6. 기능 이관 시작

## 단계 완료 기준

### 1단계 완료 기준
- 개선 계획 문서 작성 완료
- 다음 단계 작업 범위 합의 가능 상태

### 2단계 완료 기준
- 단일 파일 구조 해소
- 로컬에서 기존 핵심 기능 동작 확인

### 3단계 완료 기준
- 실제 서비스 전환용 기술 스택 제안 확정
- 다음 개발 순서 제안 완료

## 비서 의견
이 앱은 버릴 프로토타입이 아니라, 한 번 구조를 바로잡으면 충분히 다시 키울 수 있는 베이스다. 다만 지금은 기능 추가보다 정리가 먼저다. 가장 안전한 경로는:

1. 구조 분리
2. 핵심 흐름 재확인
3. 백엔드 붙이기

순으로 가는 것이다.
